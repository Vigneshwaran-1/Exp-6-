# Exp - 6 STUDENT ONLINE EXAM REGISTRATION SYSTEM

# AIM:
To create UML diagrams for Student Online Exam Registration System

# SRS (Procedure):
~~~
Problem: 
Students need a system to register for exams online. Manual registration is slow and error‑prone.

Actors: Admin, Student.

Modules:
Student → Register account, select exam, pay fees, get admit card.
Admin → Manage exams, verify students, generate reports.

Functional Requirements:
Student registration and login
Exam selection and scheduling
Fee payment and confirmation
Admit card generation

Non‑Functional Requirements:
Easy to use, secure login, reliable data storage, fast processing.
~~~
